Git commands are divided into two categories: **porcelain** and **plumbing**. Understanding this distinction helps you know when to use a high-level shortcut versus when you need precise, low-level control.

## Porcelain vs Plumbing

| Category  | Porcelain Commands               | Plumbing Commands                    |
| --------- | -------------------------------- | ------------------------------------ |
| Level     | High-level (user-friendly)       | Low-level (internal, script-friendly)|
| Audience  | End users                        | Tools, scripts, Git internals        |
| Output    | Readable, formatted              | Raw, structured (for parsing)        |
| Stability | Stable, designed for daily use   | May change, not always user-friendly |

> **Tip:** Porcelain = Pretty (what users interact with). Plumbing = Pipes (what Git is built on).

## Plumbing Commands

Plumbing commands are used by Git internally and in automation scripts. They provide fine-grained, predictable control over Git's object model.

### 10 Manipulators

Commands that **modify** Git's internal objects and structures.

| Command              | Description                                                   |
| -------------------- | ------------------------------------------------------------- |
| `git hash-object`    | Compute object ID and optionally create a blob from a file    |
| `git update-index`   | Register file contents in the working directory to the index  |
| `git write-tree`     | Create a tree object from the current index                   |
| `git commit-tree`    | Create a new commit object                                    |
| `git update-ref`     | Update the object name stored in a ref safely                 |
| `git symbolic-ref`   | Read, modify, and delete symbolic refs                        |
| `git mktree`         | Build a tree object from stdin                                |
| `git read-tree`      | Read tree information into the index                          |
| `git checkout-index` | Copy files from the index to the working directory            |
| `git prune`          | Remove unreachable objects from the object database           |
